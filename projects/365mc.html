Industry–Academia Collaboration Project

Optimizing Purchase Conversion Strategy via STT Counseling Data Analysis

Growth Hackers × 365mc  |  2025

Project Background & Objectives

365mc, a leading liposuction clinic in Korea, faced a systematic performance gap between top- and bottom-performing outbound (OB) counselors, with counseling quality entirely dependent on individual counselor skill. The project aimed to replace this intuition-driven system with a data-driven framework by:

  1. Building a custom Korean-language Speaker Diarization + STT pipeline from scratch to render raw counseling audio analytically usable
  2. Identifying the customer characteristics, counselor voice features, and verbal strategies that drive appointment conversion in OB phone consultations
  3. Deriving upselling patterns from inbound (IB) in-clinic surgical consultations

Three analytical layers were defined and operationalized independently:

  • Customer META data — what pre-call customer characteristics correlate with conversion?
  • Counselor voice features (tone, speed, pitch) — which paralinguistic features are predictive?
  • Counseling STT text — which specific verbal behaviors and response strategies drive conversion?

Theme 1. STT Pipeline Development

1-1. Problem Diagnosis

The existing vendor transcription (Clova Note) was determined unusable through systematic failure analysis:

  • Speaker diarization failure: Overlapping speech collapsed multiple speakers into a single track; brief interruption segments were particularly error-prone, making customer vs. counselor utterance separation impossible
  • Domain transcription errors: Medical brand terminology was systematically mis-transcribed due to phonetic similarity with common Korean words; the same entity was represented in multiple inconsistent forms across transcripts

These two failure modes necessitated building a full replacement pipeline rather than post-correcting the existing output.

1-2. Speaker Diarization Module

Problem framing: Global SOTA models (e.g., MossFormer2) are trained predominantly on English and multilingual corpora, creating a language gap for Korean — specifically Korean prosody, intonation contours, and overlapping speech patterns. Acquiring labeled, correctly-separated Korean counseling audio at training scale was infeasible due to privacy constraints.

Dataset construction strategy:

Rather than attempting to collect labeled real-world data, a synthetic data construction pipeline was designed:

  • Source corpora: KSponSpeech (large-scale Korean naturalistic speech corpus) + WHAM! noise dataset (standard noise augmentation corpus)
  • Synthesis algorithm: SparseKMix — selected because it generates synthetic overlapping speech with realistic conversational silence and interruption distributions, rather than naively concatenating clean speech segments
  • Output: High-quality labeled Korean overlapping speech data at training scale, with ground-truth speaker labels known by construction

Model fine-tuning:

  • Base model: MossFormer2 SS 16K
  • Applied LoRA (Low-Rank Adaptation): by freezing the pretrained weights and training only low-rank adapter matrices injected into the model, only 196,608 parameters (0.35% of total weights) required gradient updates — enabling adaptation to Korean speech and overlap characteristics within available compute constraints

Loss function redesign:

During LoRA training, output signal amplitude became unstable — the existing SI-SNR loss function measures only signal ratios and provides no constraint on absolute amplitude scale. Diagnosed as a LoRA weight scaling instability, the loss function was modified by adding an L1 Loss term:

Loss_total = -(1 - α) × SI-SNR + α × L1 Loss

The L1 term forces predicted waveforms to stay within the amplitude envelope of the ground-truth signal, and its sparsity-inducing property additionally suppresses residual noise in silent segments — improving downstream STT accuracy by reducing false activation in non-speech regions.

Inference strategy for long-form audio:

  • GPU memory constraints make processing 5+ minute calls in a single forward pass infeasible. Designed a chunked inference approach: 3-minute windows with 10-second overlap
  • Speaker label continuity across chunks is not guaranteed by default. Resolved by computing cosine similarity across the overlap region between adjacent chunks and using it to automatically realign speaker labels — preventing speaker identity flip across chunk boundaries
1-3. STT Module
  • Model selection: Whisper large-v3, chosen after comparative evaluation of Whisper-family models on transcription accuracy and timestamp stability
  • Output: Chunk-based timestamps (utterance-level start/end times per speaker), enabling all downstream turn-taking and sequence analyses. Word-level timestamps were also generated for potential future use in fine-grained overlap and interruption detection.
1-4. LLM Post-Processing Module

A two-pass LLM correction layer (OpenAI GPT-4o mini) was applied after STT to address remaining error modes:

  • First Pass (domain correction): A domain-specific lexicon was injected directly into the prompt, covering medical terminology, brand names, and phonetically similar error pairs. This corrected systematic mis-transcriptions that Whisper's general-domain training could not resolve.
  • Second Pass (turn-taking reconstruction): Timestamp information from both speakers was used to detect fragmented utterances caused by overlapping speech and reorder them into a coherent conversational sequence — reconstructing the actual turn-taking flow rather than outputting interleaved fragments.

Theme 2. OB Counseling Analysis

2-1. Customer META Analysis

Data: OB counseling records with complete customer survey responses, covering variables including area of concern, prior procedure experience, dietary habits, inquiry count, exercise habits, alcohol consumption, and weight change history.

The core question was whether conversion outcome could be predicted from pre-call customer attributes — testing the hypothesis that the customer's decision is partially pre-formed before the call begins.

  • Categorical variables: Chi-square test of independence to identify which categorical attributes showed between-group differences in conversion rate
  • Continuous variables: Mann-Whitney U test (non-parametric, appropriate given non-normal distributions)
  • Variables showing significant between-group differences were subjected to One-vs-Rest subgroup analysis to identify which specific groups drove the effect
  • Interaction effects: Logistic regression with interaction terms to test whether combinations of META attributes produced effects not explained by main effects alone

Significant variables were carried forward into the META × STT cross-analysis to test whether they also modulate how customers respond to price and risk signals during the call.

To isolate counseling behavior from individual counselor baseline differences, six features were normalized at the per-counselor level before analysis: energy_mean, energy_std, agent_cpm, articulation_rate, pause_frequency, agent_latency.

Analysis pipeline:

  1. Statistical testing: t-test or Mann-Whitney U test per feature, comparing successful vs. failed calls
  2. Logistic regression: Quantified the direction and magnitude of each feature's effect on conversion probability; features with high collinearity were removed before modeling
  3. ML-based conversion prediction: Random Forest, LightGBM, and XGBoost were trained and compared to assess predictive ceiling and generate feature importance rankings
  4. SHAP analysis: Applied to the LightGBM model to detect nonlinear relationships and threshold effects — specifically to identify whether features had monotone effects or showed optimal ranges
2-3. OB STT Text Analysis

Tagging schema design:

To convert unstructured conversational transcripts into analyzable structured data, a three-layer tagging schema was designed and applied across all OB counseling records.

Layer 1 — Topic tags (10 categories):

Segmented each call into meaning units and assigned a primary topic to each segment. Categories: Consultation Start, Customer Info, Procedure Cost, Side Effects / Risk, Discounts & Benefits, Visit Induction, Past Procedure Experience, Procedure Process, Location Guidance, Appointment Scheduling.

Layer 2 — Intent tags (5 categories):

Classified each utterance by the speaker's communicative intent: Greeting, Verify (information confirmation), Question, Inform (factual delivery), Suggestion (recommendation or action prompt). Within each major topic, the ratio of each intent type was computed separately for counselors and customers, enabling comparison of conversational dynamics in successful vs. failed calls.

Layer 3 — Flag tags (11 total: 5 customer + 6 counselor):

Customer flags — specific negative or commitment signals:

schedule_unavailable commit_date reserve_quit price_sensitive risk_worry

Counselor flags — specific persuasion and closing behaviors:

appointment_offer appointment_confirmed convince_deadline convince_promotion convince_changedate convince_free

Analysis 1 — Intent distribution by Topic

For each of the three key topics (Procedure, Risk, Price), computed counselor and customer intent ratios separately for successful and failed calls. The difference in intent ratio (success minus failure) identified which communicative behaviors were overrepresented in each outcome group. The hypothesis was that counselors in successful calls actively shape the conversational dynamic differently within each topic — not just saying different things, but using different conversational moves.

Analysis 2 — Response strategy by negative customer flag

Confirmed that negative customer flags occur at comparable frequencies in both successful and failed calls — establishing that the counselor's response to the flag, not the flag's mere presence, is the operative variable. For each negative flag type, analyzed the distribution of counselor flags appearing immediately after the negative flag, comparing success and failure groups. Additionally:

  • Computed the ratio of each topic transition following each negative flag
  • Performed log-odds ratio analysis on keyword n-grams in the counselor utterance immediately following the flag, to identify lexical patterns associated with successful vs. failed responses

For the risk_worry flag specifically, applied SBERT sentence embeddings to cluster customer risk-concern utterances into semantic sub-topics (Pain, Safety/Fear, Efficacy, Recovery, Area/Condition, Skin/Scar), then separately analyzed counselor response patterns within each sub-topic — identifying topic-specific successful and failed verbal strategies through n-gram and log-odds ratio analysis.

Analysis 3 — Appointment proposal timing

Tested whether the timing of the counselor's first appointment proposal (measured by number of topics covered prior to the proposal) affected conversion. Also tested whether price discussion prior to the first appointment proposal moderated conversion rates. Used the appointment_offer, appointment_confirmed, and topic sequence data to reconstruct proposal timing for each call. Analyzed conversion rates as a function of (a) total proposal count, (b) build-up topic count before first proposal, and (c) whether a Price topic appeared before the first proposal.

Analysis 4 — Flag-based Decision Tree

Trained a Decision Tree on counselor flag counts as features with conversion as the target variable. The objective was to identify which specific counselor behaviors most discriminate between converted and non-converted calls, and at what frequency thresholds the effect is largest. The Decision Tree's interpretable split structure allowed identification of the most globally impactful counselor behaviors (those appearing at the root or early splits) and enabled quantification of the conversion rate lift associated with each behavior.

2-4. META × STT Cross-Analysis

Combined customer META attributes with per-call flag occurrence data to test whether different customer segments (defined by META) generate different rates of negative signals and respond differently to those signals when they occur.

  • Kruskal-Wallis test to assess whether flag occurrence frequency differed significantly across META-defined groups
  • For META variables showing significant group differences in flag frequency, further analyzed post-flag conversion rates by group — specifically how much conversion dropped when each flag appeared, and whether the magnitude of that drop varied by customer segment

The goal was to identify whether the same counselor strategy was appropriate across all customers, or whether META information should modulate the counseling approach when specific flags arise.

Theme 3. IB (In-Clinic) Consultation Analysis

3-1. IB META — Upselling Analysis

Data: In-clinic surgical consultation records with complete pre-visit survey data and post-consultation surgical outcome records

Upselling operationalization: Defined upselling as any surgical procedure performed on a body area not indicated as an area of concern in the customer's pre-visit survey. Computed by comparing the set of survey-indicated concern areas against the set of actual surgical areas per patient, with the non-overlapping surgical areas classified as upsell outcomes.

Analysis design:

  • BMI: Continuous variable; analyzed upselling rate and average number of upsell areas across BMI quantile groups
  • Occupation: Categorical variable; computed average upsell area count per occupation group and ranked
  • Primary concern area → upsell area transition: Constructed a transition rate matrix from concern area to upsell area to identify which concern area combinations most frequently led to upselling on additional areas
  • Hospital selection criterion: Chi-square / proportion test comparing upselling rates across customers who cited different primary selection factors, with p-value adjustment for multiple comparisons
3-2. IB STT — Negative Signal Response Strategy Analysis

Data: Preprocessed IB STT records (post-noise filtering; IB audio quality issues prevented full-pipeline STT application to the complete dataset)

The hypothesis was that optimal response strategies exist for each type of customer negative signal, and that current counselors were not systematically applying them.

  1. SBERT sentence embeddings: Encoded all customer and counselor utterances into dense semantic vectors, enabling semantic similarity-based grouping without relying on exact keyword matching
  2. Negative signal classification: Combined keyword-based rules with semantic similarity to classify customer utterances as negative signals, and further categorized them by signal type (price resistance, side effect concern, general hesitation, etc.)
  3. Counselor response type classification: Classified the counselor's response utterance(s) following each negative signal into response strategy categories (empathy/reassurance, value proof, benefit loss framing, price concession, expertise demonstration)
  4. Response strategy matrix construction: For each (negative signal type, counselor response type) pair, computed the proportion of successful outcomes — yielding a matrix that directly quantifies the conversion lift or penalty associated with each response strategy for each signal type
  5. Logistic regression for statistical validation: Fit a logistic regression model with signal type × response strategy interaction terms to estimate odds ratios for each response strategy, establishing which effects were statistically significant

4. Conclusions & Recommendations

As PM, translated analytical outputs into actionable recommendations across four dimensions:

  • Survey instrument redesign: Identified specific limitations of the existing OB pre-call survey and proposed additional question structures to capture customer intent-timing information, enabling counselors to use survey data actively during calls
  • Counselor training guidelines: Structured the STT findings into topic-specific and flag-specific behavioral guidelines, including concrete verbal strategy examples derived from the n-gram and log-odds ratio analyses
  • IB data collection standards: Diagnosed the root causes of IB audio quality issues and specified structured recording protocols that would make future IB audio suitable for STT pipeline application
  • IB diarization extension roadmap: Identified the technical constraints preventing current speaker diarization from working on IB audio (fixed 2-speaker assumption vs. multi-party IB sessions) and proposed dataset construction and model extension approaches to resolve them
